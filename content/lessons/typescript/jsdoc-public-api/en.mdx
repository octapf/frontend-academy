# JSDoc on public APIs

## What you’ll take away

Comment at friction points: non-obvious params and possible errors.

## Details

Don’t duplicate what the type already says unless it adds semantics.

## Recommended practice

- Write down **one real case** from your repo where this topic showed up (or should have).
- When relevant, use **DevTools** (network, console, performance, layout) to connect the idea to what you can observe.

## Common pitfalls

- Copying snippets without adapting team conventions (names, layers, tests, accessibility).

## Resources

- [MDN Web Docs](https://developer.mozilla.org/en-US/)
- Official docs for React, TypeScript, or your bundler, depending on the topic.
