# `readonly` in TypeScript

`readonly` on properties and tuples signals intent: **do not mutate** that reference from consumers. It does not freeze nested objects (not a deep freeze), but it prevents classic bugs like `.sort()` mutating the original array.

```ts
type Props = { ids: readonly string[] };

function List({ ids }: Props) {
  return ids.map((id) => <li key={id}>{id}</li>);
}
```

## `ReadonlyArray<T>`

Equivalent to `readonly T[]`. Useful for public library APIs and presentational components.
