# Subscriptions and shallow comparison

## What you’ll take away

`useStore(selector)` triggers renders when the selector result changes by reference.

## Details

For partial objects use `shallow` or selectors that return stable primitives.

## Recommended practice

- Write down **one real case** from your repo where this topic showed up (or should have).
- When relevant, use **DevTools** (network, console, performance, layout) to connect the idea to what you can observe.

## Common pitfalls

- Selector returning a fresh object literal each call and forcing constant re-renders.
