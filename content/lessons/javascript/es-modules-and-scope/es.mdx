# Módulos ES: import, export y scope de archivo

## Qué vas a entender

`import` vive en el scope superior del módulo: no podés condicionarlo como un `require` clásico sin extraer a otro módulo.

## Detalle

Los imports estáticos ayudan al análisis estático; los dinámicos `import()` habilitan code splitting explícito.

- Anotá **un caso real** de tu repo donde este tema apareció (o debería haber aparecido).
- Si aplica, usá **DevTools** (red, consola, performance, layout) para conectar el concepto con lo observable.
