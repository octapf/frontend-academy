# Criterios de aceptación y Definition of Done

## Qué vas a entender

Un criterio aceptable es observable: “usuario ve error X cuando falla Y”, no “queda lindo”.

## Detalle

La DoD incluye calidad transversal: tests mínimos, telemetría, accesibilidad según convención del equipo.

## Práctica recomendada

- Anotá **un caso real** de tu repo donde este tema apareció (o debería haber aparecido).
- Si aplica, usá **DevTools** (red, consola, performance, layout) para conectar el concepto con lo observable.

## Errores comunes

- Confundir estimación puntual con compromiso sin riesgos explícitos.

## Recursos

- [MDN Web Docs](https://developer.mozilla.org/es/)
- Documentación oficial del stack que estés usando.
