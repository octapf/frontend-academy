# Clear boundaries

Between **API and UI**, the contract is the data shape plus evolution rules. Without an explicit contract, each deploy can silently break the other side.

## Useful practices

- **Schema validation** at the edge (zod, etc.) before touching global state.
- **Versioning** or well-documented optional fields when the backend outpaces the UI.
- **Normalized errors** (status + code + safe message) so the UI can render consistently.
