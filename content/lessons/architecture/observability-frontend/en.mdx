# Client-side observability

**Observability** is not “more console.log”: it is **actionable signal** when something fails or degrades: normalized errors, Web Vitals metrics, and correlation with the backend (trace id).

## Good practices

- **One** error shape in UI + send to a collector (no PII).
- **Sample** noisy events on high-traffic pages.
- **Release version** on every report so you know which deploy broke things.
