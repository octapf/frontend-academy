# Rebase vs merge: historia legible

## Qué vas a entender

Rebase interactivo limpia commits ruidosos antes del PR; requiere coordinación si la rama es compartida.

## Detalle

Merge conserva el contexto de integración; útil cuando varios equipos tocaron la misma base.

## Práctica recomendada

- Anotá **un caso real** de tu repo donde este tema apareció (o debería haber aparecido).
- Si aplica, usá **DevTools** (red, consola, performance, layout) para conectar el concepto con lo observable.

## Errores comunes

- Rebasear sobre main sin actualizar local y generar conflictos masivos a último momento.
