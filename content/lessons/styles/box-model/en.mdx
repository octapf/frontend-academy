# Box model

```css
.card {
  width: 200px;
  padding: 16px;
  border: 2px solid #ccc;
  margin: 8px;
}
```

Every element is a **box** with layers: content, padding, border, and margin. Default `width` (`content-box`) usually measures content only; padding and border add outside that number.

## `box-sizing`

```css
*,
*::before,
*::after {
  box-sizing: border-box;
}

.panel {
  width: 320px;
  padding: 24px;
  border: 1px solid #e4e4e7;
}
```

With `box-sizing: border-box`, `width` includes **padding and border**, which makes responsive layouts easier: declared width matches what you budget in a row.

```css
h2 {
  margin-top: 1.5rem;
}
p {
  margin-bottom: 1.5rem;
}
/* Two adjacent vertical margins may collapse into one */
```

<Callout title="Margin collapse">
Between adjacent block boxes, vertical margins can **collapse**: the final gap is not always the sum of both.
</Callout>

## Measurements as text

With design tokens or UI props you sometimes validate strings like `"16px"` before converting to a number for layout math (spacing, motion). A small strict parser avoids silent `NaN` values.
