# Native forms and labels

## What you’ll take away

The label `for` must match the input `id`; in dynamic lists use stable ids or `aria-labelledby`.

## Details

Tie validation messages with `aria-describedby` so assistive tech reads context.

## Recommended practice

- Write down **one real case** from your repo where this topic showed up (or should have).
- When relevant, use **DevTools** (network, console, performance, layout) to connect the idea to what you can observe.

## Common pitfalls

- Simulating a checkbox with `div` and `onClick` without role or keyboard support.
