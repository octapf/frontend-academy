# Error Boundary vs errores async (Query)

## Qué vas a entender

Los Error Boundaries no capturan event handlers ni promesas no observadas: los errores de `useQuery` se modelan con estado `error`.

## Detalle

Componé: boundary para render tree, `onError`/`throwOnError` según librería para datos remotos.

## Práctica recomendada

- Anotá **un caso real** de tu repo donde este tema apareció (o debería haber aparecido).
- Si aplica, usá **DevTools** (red, consola, performance, layout) para conectar el concepto con lo observable.

## Errores comunes

- Esperar que un boundary salve un `fetch` mal manejado dentro de un effect sin `await`/`catch`.
