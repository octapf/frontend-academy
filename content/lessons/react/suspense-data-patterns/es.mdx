# Suspense y datos (patrones)

## Qué vas a entender

Suspense agrupa regiones que pueden “esperar” recursos; el fallback debe ser **accesible** y no parpadear sin control.

## Detalle

Definí qué capa posee el fetch y cómo se invalida el cache para evitar estados imposibles.

## Práctica recomendada

- Anotá **un caso real** de tu repo donde este tema apareció (o debería haber aparecido).
- Si aplica, usá **DevTools** (red, consola, performance, layout) para conectar el concepto con lo observable.
